# FastCli 手动打包说明

## 目的

本文档用于说明如何在本机手动打包 `FastCli`，生成以下两类交付物：

- 完整可执行程序：`FastCli.exe`
- 在线补装运行环境的安装包：`FastCli-Setup-v1.0.0.exe`

当前项目的 `build/` 目录只保留两个打包入口：

- `build/package-exe.bat`
- `build/package-setup.bat`

以及一个安装脚本依赖：

- `build/FastCli.Setup.iss`

## 环境要求

### 必需环境

- Windows
- `.NET 8 SDK`

检查命令：

```powershell
dotnet --version
```

### 安装包环境

如果需要生成 `setup.exe`，还需要安装 `Inno Setup 6`。

默认使用的编译器路径通常是：

```powershell
C:\Users\48565\AppData\Local\Programs\Inno Setup 6\ISCC.exe
```

如果本机没有安装，可以执行：

```powershell
winget install --id JRSoftware.InnoSetup -e --silent --accept-package-agreements --accept-source-agreements
```

## 版本号

版本号统一以 `FastCli.Desktop/FastCli.Desktop.csproj` 中的 `<Version>` 为唯一维护入口。

当前已经收敛为：

- `AssemblyVersion = $(Version).0`
- `FileVersion = $(Version).0`
- `InformationalVersion = $(Version)`

也就是说，发版时只需要修改：

- `FastCli.Desktop/FastCli.Desktop.csproj` 中的 `<Version>`

## 1. 生成完整 exe

在项目根目录执行：

```powershell
.\build\package-exe.bat
```

输出目录：

```text
artifacts/release/FastCli-v版本号-win-x64-portable/
artifacts/release/FastCli-v版本号-win-x64-portable.zip
```

说明：

- 这是 self-contained 便携独立包
- 包内会包含 `FastCli.exe` 以及依赖的 `.dll`
- 当前不能再使用单文件发布，因为终端控件依赖 `Microsoft.Terminal.Control.dll` 这类原生库，单文件模式下会出现 `Dll was not found`
- 最终交付推荐直接使用 zip 包
- 目标机器无需额外安装 `.NET Desktop Runtime`

## 2. 生成 setup 安装包

在项目根目录执行：

```powershell
.\build\package-setup.bat
```

输出目录：

```text
artifacts/release-setup/FastCli-Setup-v版本号.exe
```

说明：

- 这是 framework-dependent 安装包
- 安装内容来自 `artifacts/release-setup/app/` 下的发布目录
- 更适合 `WPF + SQLite` 这类有原生依赖的桌面程序
- 如果目标机器缺少 `.NET Desktop Runtime 8`，安装器会自动按顺序尝试以下下载地址：
  - Microsoft 官方短链 `aka.ms`
  - Microsoft 官方直链 `builds.dotnet.microsoft.com`
  - GitHub Releases 镜像地址
- 安装器检测到缺少 runtime 后会直接自动下载并安装，不再二次确认
- 下载阶段会显示来源、文件名和下载进度
- 安装阶段会显示运行环境安装状态
- 如果本次安装过程中刚补装了 runtime，安装完成后不会自动拉起 FastCli，请手动启动一次

当前默认第三层兜底地址：

```text
https://github.com/mostbean-cn/fast-cli/releases/download/runtime-cache/windowsdesktop-runtime-8.0.25-win-x64.exe
```

## 3. 当前相关文件

- 项目打包配置：`FastCli.Desktop/FastCli.Desktop.csproj`
- 安装脚本：`build/FastCli.Setup.iss`
- 完整 exe 打包入口：`build/package-exe.bat`
- setup 打包入口：`build/package-setup.bat`
- 应用图标：`assets/FastCli.ico`

## 4. 常见问题

### 问题 1：`FastCli.exe` 被占用，`dotnet publish` 失败

典型报错：

```text
The process cannot access the file '...\FastCli.exe' because it is being used by another process.
```

处理方式：

1. 先关闭正在运行的 `FastCli.exe`
2. 重新执行打包脚本

说明：

- 现在 `package-exe.bat` 会先发布到独立 staging 目录，再生成便携目录和 zip，已经明显降低旧产物占用导致失败的概率

### 问题 2：完整 exe 体积很大

这是正常现象，因为 `package-exe.bat` 生成的是 self-contained 多文件便携包，会把 `.NET Desktop Runtime` 和原生依赖一起带上。

### 问题 3：启动时报 `Dll was not found`

如果你看到这类报错，通常是因为用了旧的单文件产物，或者手动拷贝时漏掉了旁边的原生 `.dll`。

处理方式：

1. 删除旧的 `artifacts/release/FastCli-v版本号-win-x64-portable/`
2. 重新执行 `build/package-exe.bat`
3. 不要只拷贝单个 `FastCli.exe`，而是整体分发整个目录或 zip

### 问题 4：setup 安装时需要联网

这是当前设计使然。`package-setup.bat` 生成的是在线补装 runtime 的安装包。

如果三层下载地址都不可用，安装器会提示下载失败。

## 5. 推荐发布顺序

每次手动发版建议按以下顺序：

1. 修改 `<Version>`
2. 执行 `build/package-exe.bat`
3. 手动验证 `artifacts/release/FastCli-v版本号-win-x64-portable.zip`
4. 执行 `build/package-setup.bat`
5. 最终交付：
   - `artifacts/release/FastCli-v版本号-win-x64-portable.zip`
   - `artifacts/release-setup/FastCli-Setup-v版本号.exe`
