# FastCli 手动打包说明

## 目的

本文档用于说明如何在本机手动打包 `FastCli`，生成以下两类交付物：

- 单文件可执行程序：`FastCli.exe`
- 安装包：`FastCli-Setup.exe`

当前项目采用：

- `dotnet publish` 生成单文件 `exe`
- `Inno Setup` 生成安装包

## 环境要求

### 必需环境

- Windows
- `.NET 8 SDK`

检查命令：

```powershell
dotnet --version
```

### 安装包环境

如果需要生成 `setup.exe`，还需要安装 `Inno Setup 6`。

当前项目默认使用的编译器路径是：

```powershell
C:\Users\48565\AppData\Local\Programs\Inno Setup 6\ISCC.exe
```

如果本机没有安装，可以执行：

```powershell
winget install --id JRSoftware.InnoSetup -e --silent --accept-package-agreements --accept-source-agreements
```

## 打包前说明

### 图标与版本号

以下信息已经在项目文件中配置：

- 应用图标：`assets/FastCli.ico`
- 产品名：`FastCli`
- 版本号：`1.0.0`

配置位置：

- `FastCli.Desktop/FastCli.Desktop.csproj`

### SQLite 初始化

数据库初始化 SQL 已内嵌到程序中，不依赖外部 `sql/001_init.sql` 文件。

所以打出来的单文件 `exe` 可以独立运行，不需要额外携带 SQL 文件。

## 1. 生成单文件 exe

在项目根目录执行：

```powershell
dotnet publish .\FastCli.Desktop\FastCli.Desktop.csproj `
  -c Release `
  -r win-x64 `
  --self-contained true `
  -p:PublishSingleFile=true `
  -p:EnableCompressionInSingleFile=true `
  -p:IncludeNativeLibrariesForSelfExtract=true `
  -p:DebugType=None `
  -p:DebugSymbols=false `
  -o .\artifacts\release
```

输出目录：

```text
artifacts/release/FastCli.exe
```

## 2. 生成安装包 setup.exe

在已经生成好 `artifacts/release/FastCli.exe` 之后，执行：

```powershell
& 'C:\Users\48565\AppData\Local\Programs\Inno Setup 6\ISCC.exe' .\build\FastCli.Setup.iss
```

输出目录：

```text
artifacts/release/FastCli-Setup.exe
```

## 3. 如果想打到新目录

当旧目录中的 `FastCli.exe` 正在被占用，无法覆盖时，不要强删，直接打到新的输出目录。

例如打到 `artifacts/release-v2`：

### 先生成新的 exe

```powershell
dotnet publish .\FastCli.Desktop\FastCli.Desktop.csproj `
  -c Release `
  -r win-x64 `
  --self-contained true `
  -p:PublishSingleFile=true `
  -p:EnableCompressionInSingleFile=true `
  -p:IncludeNativeLibrariesForSelfExtract=true `
  -p:DebugType=None `
  -p:DebugSymbols=false `
  -o .\artifacts\release-v2
```

### 再生成指向该目录的安装包

```powershell
& 'C:\Users\48565\AppData\Local\Programs\Inno Setup 6\ISCC.exe' `
  /DMyAppSourceExe='..\artifacts\release-v2\FastCli.exe' `
  /DMyOutputDir='..\artifacts\release-v2' `
  .\build\FastCli.Setup.iss
```

输出结果：

- `artifacts/release-v2/FastCli.exe`
- `artifacts/release-v2/FastCli-Setup.exe`

## 4. 常见问题

### 问题 1：`FastCli.exe` 被占用，`dotnet publish` 失败

典型报错：

```text
The process cannot access the file '...\FastCli.exe' because it is being used by another process.
```

处理方式：

1. 先关闭正在运行的 `FastCli.exe`
2. 如果不方便关闭，直接改输出目录，例如 `release-v2`

### 问题 2：双击 exe 没反应

先确认你运行的是最新打包文件，而不是旧目录中的历史版本。

建议优先使用：

- `artifacts/release/FastCli.exe`
- 或你最新指定的输出目录版本

### 问题 3：安装包找不到源文件

说明 Inno Setup 在编译时找不到要打包的 `FastCli.exe`。

排查顺序：

1. 先确认 `dotnet publish` 已成功
2. 确认 `artifacts/release/FastCli.exe` 确实存在
3. 如果打到自定义目录，使用带 `/DMyAppSourceExe` 的命令

## 6. 推荐发布顺序

每次手动发版建议按以下顺序：

1. 修改版本号
2. 执行 `dotnet publish`
3. 手动双击验证 `FastCli.exe`
4. 执行 `ISCC.exe` 生成安装包
5. 最终交付：
   - `FastCli.exe`
   - `FastCli-Setup.exe`
